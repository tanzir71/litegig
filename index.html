A compact workflow for real handoffs.

LiteGig keeps the app focused on the operational path: create a request, let a runner accept it, record pickup, confirm payment, mark delivery, rate the other side, and export the history.

Request queue

Filter by status, task type, distance, and keyword search across titles, descriptions, people, and metadata.

Schema-driven jobs

Admins define task types with fields for addresses, prices, files, select options, dates, numbers, and notes.

Custody timeline

Each request shows where it started, who currently owns the work, and which action should happen next.

Clear hosting responsibilities.

The repository can serve a static public layer and a PHP app package. Keep those responsibilities separate so deployment notes stay honest.

Static site

index.html and docs.html are navigable pages for repository hosting or GitHub Pages.

Interactive app

litegig.php is the product entrypoint and needs a PHP 8 host with PDO SQLite enabled.

Deploy bundle

The shared-host package is optional but complete: PHP file, env example, docs, and security scan tools.

Start with sample data, then harden for production.

Register the first account as admin, load sample data from the admin nav, and use the setup and security checklists before putting real users on it.

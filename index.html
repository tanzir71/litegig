Live state

Built for coordinators who need the next move to be obvious.

LiteGig keeps the operational path explicit: create a request, assign or let a runner accept it, record pickup, confirm payment, mark delivery, rate the other side, and export the history.

Queue and Open Pool

Filter operational work by status, task type, distance, fee, and keyword search across titles, descriptions, people, and metadata.

Schema-driven jobs

Admins define task types with fields for addresses, prices, files, select options, dates, numbers, and notes.

Custody timeline

Each request shows where it started, who currently owns the work, and which action should happen next.

One production direction: PHP and SQLite on infrastructure people already have.

The repository packages the working app for shared hosting. Static pages explain the product; the PHP package serves the real queue.

Shared hosting

Run npm run deploy:shared, upload the zip, extract it in cPanel, and create the first real admin.

Interactive app

litegig.php is the product entrypoint and needs a PHP 8 host with PDO SQLite enabled.

Low-cost operations

SQLite keeps startup cost low, needs no managed database, and fits simple file backup workflows.

Create real admins, rotate seed access, and back up before inviting users.

The Admin Console can create production admins, reset passwords, disable accounts, and audit every access change with reason and before/after snapshots.

Quick start

Use a PHP 8 build with pdo_sqlite enabled. Runtime data is intentionally ignored by Git: local env files, SQLite databases, logs, and upload folders.

npm run deploy:shared
cp .env.example .env
php -S 127.0.0.1:8080

Open http://127.0.0.1:8080/litegig.php, register the first user, and that account becomes admin automatically.

Run syntax and security checks on the same PHP version family that will handle real operators.

First deploy and credential rotation

Create a real production admin before disabling any seed or bootstrap admin. After the new admin logs in successfully, disable seed access, rotate old passwords, confirm old passwords fail, and run a verified backup.

Seed requesterrequester@example.test must not remain usable in production.
Seed runnerrunner@example.test must not remain usable in production.

The Admin Console prevents disabling the current active admin session and prevents disabling the last active admin. Every access change requires a reason and writes before/after snapshots to the audit log.

Operator workflow

LiteGig models a local request as a custody path. The detail page shows who opened the job, who currently owns the next move, what action should happen next, and any private proof files attached to updates.

01New
02Accepted
03Picked up
04Paid
05Delivered
06Completed

Manual payment confirmation remains the default. Optional gateway adapters can record references and signed webhook confirmations without storing card details.

Deployment model

The production backend is PHP + SQLite. Keep demos and static pages separate from the real queue:

  • Static repository pages: serve index.html, docs.html, and styles/tokens.css as public landing and documentation pages.
  • Maintainer demo: vercel-demo/ is static preview material only and is deployed with npm run maintainer:deploy:demo.
  • Interactive app host: serve litegig.php, PWA files, and brand assets on PHP 8+ with pdo_sqlite, sqlite3, and fileinfo.
  • Shared-host bundle: run npm run deploy:shared, upload/extract the zip, copy .env.example to .env, and verify health.php.
  • VPS/SSH: package locally, then rsync -avz dist/shared-host/litegig/ user@host:/var/www/litegig/.
  • Vercel production: requires a native port because the current backend runtime is PHP + SQLite.

LLM-assisted deployment agents can use the copy-paste prompt in README.md or SETUP.md.

npm run deploy:shared
php tools/maintenance.php --apply --backup
php tools/production_audit.php

Security expectations

The app includes output escaping, CSRF checks for state-changing forms, prepared SQL statements, server-side ownership checks, queued email/SMS notifications with user preferences, hashed delivery OTPs, signed payment webhooks, session regeneration, private attachment downloads, upload validation, rate limits, CSP/security headers, health JSON, cron backups, and env-based runtime config.

  • Keep .env, databases, logs, and uploads outside public links.
  • Set a long random LITEGIG_CRON_TOKEN before enabling HTTP cron access.
  • Force HTTPS at the domain level and keep PHP/cPanel extensions patched.
  • Back up the SQLite database daily with php tools/maintenance.php --apply --backup.
  • Use provider feature flags for SMS/payment/error adapters; log-only/manual modes are valid low-infrastructure defaults.

Release and operations files: docs/release-checklist.md, docs/mysql-migration.md, docs/decisions/, and monitoring/uptime.example.yml.

Troubleshooting

SQLite driver missing

Enable pdo_sqlite in the PHP extension manager or deploy to a PHP host that supports SQLite.

Uploads fail

Check LITEGIG_UPLOAD_DIR, directory permissions, file size limits, and allowed file extensions.

Sample data cannot load

Confirm you are logged in as an admin. The first registered user is admin; later users are not.

CSV excludes emails

That is the safe default. Admin PII export requires the env flag to be enabled and an explicit export request.

Roadmap

  • Keep PHP + SQLite as the production default until real write concurrency requires a database change.
  • Keep maintainer-only demos clearly separate from production deployment docs.
  • Connect production SMS and payment providers through the existing webhook adapters.
  • Consider a native Vercel or managed-stack port only as a separate product direction.
